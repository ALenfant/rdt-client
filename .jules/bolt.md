## 2024-05-19 - Optimize Database Updates in Entity Framework
**Learning:** Found multiple places in `DownloadData.cs` and `TorrentData.cs` where entities are being retrieved into memory and then modified instead of using EF Core's efficient `ExecuteUpdateAsync` or `ExecuteDeleteAsync`. The instructions already mention this as a critical learning: "When making database updates or deletions in Entity Framework Core, use `ExecuteUpdateAsync` (chaining `.SetProperty()` if updating multiple fields simultaneously) and `ExecuteDeleteAsync` instead of fetching the entire entity first to improve performance and avoid tracking overhead." I will use this performance tip to convert `DownloadData.cs` methods to use `ExecuteUpdateAsync`.
**Action:** When updating a single field, switch to `ExecuteUpdateAsync`. This skips loading tracking entity and avoids an extra round trip.
