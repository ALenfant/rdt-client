## 2024-05-20 - Array Sorting Accessors (Schwartzian Transform)
**Learning:** In highly reactive apps (like this Angular client updating via WebSocket/polling), sorting arrays dynamically inside templates or component subscriptions can trigger an O(N log N) evaluation of accessor functions. If the accessor does expensive string parsing (`.split('.').reduce()`) or derives values using loops (like `getTorrentStatus`), it blocks the main thread noticeably on large arrays.
**Action:** When a sort uses computed accessors, implement a Schwartzian transform (`array.map(i => ({i, val: accessor(i)})).sort(...).map(w => w.i)`) to guarantee the accessor is evaluated exactly O(N) times instead of O(N log N). Additionally, fast-path dot-notation property accessors by avoiding `.split` if `.includes('.')` is false.
